# Agent Guide

## 文档优先级

1. `MVP_SPEC.md`
2. `MVP_ARCHITECTURE.md`
3. `MVP_IMPLEMENTATION_CHECKLIST.md`
4. `README.md`

## 工作约定

- 先满足 MVP 闭环，不主动扩 scope
- 玩法规则、数值、验收场景只以 `MVP_SPEC.md` 为准
- 代码组织与实现边界按 `MVP_ARCHITECTURE.md`
- 当前完成状态以代码事实为准，清单只反映状态
- 修改玩法规则时，必须同步更新 `MVP_SPEC.md`
- 完成功能后，必须同步更新 `MVP_IMPLEMENTATION_CHECKLIST.md`
- 若当前状态变化明显，也应同步更新 `README.md`
- 若文档冲突，玩法按 spec，代码组织按 architecture
